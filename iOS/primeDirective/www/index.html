Week 1

Menu

1. Less obvious perks of using 'live' data resources include logins and payments. How could Twitter be used in these two scenarios?

Using Twitter as a login portal can make for a better user experience, taking less time and getting the user to using your app quickly. Using the API to make payments it speeds up the process of purchasing an item you want, and we all know that convenience can increase sales.

2. Visit Programmableweb.com and search for JSON-P data sources. Why would we need JSON-P, specifically?

JSON-P is JSON with padding. I gives us a method of requesting data from different domains helping us around the "same origin policy". This ability to retrieve data from different sites makes a richer more dynamic applications.

3. Based on the JSON-P search results: what data resource(s) do you think might pair well with which types of Native device features?

I am very interesting in using feed from Instagram and Tumbler, and pairing it with the native device features such as the camera and globalization. The knowledge of how to implement these mashup features into an application would be would be great.

4.Please describe any design and functionality intentions you may have for your Demo app components this month.

I have always like the use of navigational panels, and I have implemented them into my application. I think panels are a great way to make applications more accessible when there are a lot of options for the user, panels can give the user an easy way to navigate on a small screen. My main page just has text on it. I am working on making this more interesting with some sort of grid layout, with images and text.

5. Briefly describe some past 'lessons learned' regarding workflow as it relates to research, design, development, etc.

Don't be afraid to fail. I need to give myself room to try different things without freaking out about screwing everything up.

Week 2

Menu

1. What are some of the ways that you could change your HTML on your Demo app to be more accessible? (Remember, you can use the Section tag with jQuery, for instance)

Header Tags, title tags for images and a simple, well organized layout for the app would make it more accessible to visually impaired users that might need to use a screen reader. Listing the items in a list alphabetically would seem to help as well.

2. Visit a video service on the web (like YouTube) and search for "Screen Reader Demo" and "Mobile Screen Reader" - what are some of the common frustrations (or common support mechanisms) that you see among the examples that are given - how could you fix them (or use them)?

Issues that arise are that screen readers still sound very robotic and some of them can be very expensive. Another possible issue a user might have is the number of links on a crowded website. A site can have several hundred links on it and the user would have to know what they are looking for and how to find it to maneuver though given site. Designing a site that has a great organizational layout with appropriate headings and a minimal number of links would help make it more accessible.

3. Devices: Please list two devices (and some of their features) that are made specifically to support Accessibility.

Apple has done a lot to make iOS accessible to people with disabilities. Right now it's leading as the preferred smartphone and tablets for users that need more accessibility but Android is not far behind though. Some of the accessibility features in iOS include VoiceOver and Siri for the vision impaired. For the hearing impaired there are features like FaceTime for calls and TTY support and visual voicemail. The features I listed a just a few of the what iOS offers for accessibility. The Emporia Click is a phone designed from the ground up for accessibility, specially for the elderly and its very easy the use. The down side of the Emporia Click is that its more of a feature phone. With large buttons and hearing aid-compatible speakers and its simplified UI, this is an appropriate phone for it's target audience.

4. Mobile Apps: Please list two apps (either Android or iOS, or both) that are made specifically to support Accessibility.

There is an iOS game called Slug Bugs. It is created specifically for headphone-wearing gamer. The 3D binaural audio allows one to hear 3D sounds coming from any direction. The localized sound makes games accessible to the visually impaired. List Recorder for iOS is also another app built with accessibility in mind. It enables both the visually-impared and sighted users to create lists of times that can contain text and audio. One can record a quick note, include a text memo and give an item a title. All the user needs to do is choose the type of format that suites their needs.
